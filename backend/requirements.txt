# numpy 2.4 raised the baseline of its x86_64 wheels to x86-64-v2 (SSE4.2/POPCNT). A
# Home Assistant box is very often a VM whose CPU model is generic — Proxmox's default
# `kvm64` is x86-64-v1 — and there numpy 2.4+ dies on import with "NumPy was built with
# baseline optimizations (X86_V2) but your machine doesn't support (X86_V2)".
#
# That failure is not recoverable either: numpy's C extension is single-phase-init, so
# once its first import dies the .so is already loaded and EVERY retry raises "cannot
# load module more than once per process" for the life of the process. The stocks app is
# then permanently broken with an error that names nothing useful.
#
# 2.3.x is the last series whose wheels run on those CPUs, and yfinance/pandas are happy
# with it. Unpin when a numpy ships that keeps the older baseline (or when arm64-only).
numpy<2.4
